# Phase5 - データ基盤・統計・DS連携

## このphaseのゴール

**DSP案件のデータ基盤技術を「会話の中で迷子にならない」レベルまで理解すること。**

コードを書くフェーズではなく、「何をしているか・なぜそれを使うか」が
DS/データエンジニアとの会話で理解できる状態を目指す。

---

## 背景（なぜこのphaseが必要か）

案件では以下のメンバーと連携することが想定される。

- データサイエンティスト → ターゲティングロジック・シミュレーションモデルを作る
- データエンジニア → Databricks / BigQuery / dbt でデータパイプラインを管理
- フロントエンドエンジニア → そのモデルの結果を API 経由で受け取り、UI に表示する

「モデルがどう動くか」を実装する必要はないが、
「何を入力して何が返ってくるか」「なぜその値が出るか」は理解している必要がある。

---

## 完了条件（phase5 全体）

- [ ] Databricks が何をするツールか・Airflow と何が違うか説明できる
- [ ] Delta Lake と通常のデータレイクの違いを説明できる
- [ ] BigQuery / Snowflake / Databricks の使い分けが説明できる
