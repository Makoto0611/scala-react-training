# 課題2 - コンテンツパフォーマンスダッシュボード

**目安時間**: 6〜8時間
**参照リソース**: react.dev / TypeScript公式

step1〜3で学んだ全ての知識を統合して、動く成果物を完成させる。

---

## 仕様

`src/project3-dashboard/` に実装する。

### 表示内容

```
[コンテンツパフォーマンスダッシュボード]

フィルター: [コンテンツID入力欄____] [クリア]
ソート: [視聴数] [いいね数] [いいね率]

| コンテンツID | 視聴数 | いいね数 | いいね率  |
|------------|--------|---------|---------|
| ct_001     | 1,000  | 50      | 5.00%   |
| ct_002     |   800  | 12      | 1.50%   |

※ クリックで選択・再クリックで解除（行をハイライト）
```

### データソース

phase1の課題1で生成した `contents.json` を参照する。
フォーマットは以下の通り：

```json
[
  {"contentId": "ct_001", "views": 2, "likes": 1, "likeRate": 50.00}
]
```

### レコメンドボタン（phase3の繋ぎこみ用）

各行に「レコメンドする」ボタンを追加する。
この段階ではクリックしても `console.log` するだけでよい。
phase3で Scala API との繋ぎこみを実装する。

---

## コンポーネント分割要件

最低でも以下のコンポーネントに分割すること：

```
App.tsx                → データ取得・状態管理
FilterInput.tsx        → テキストボックス・クリアボタン
SortButtons.tsx        → ソートキー選択ボタン
ContentTable.tsx       → テーブル全体
ContentTableRow.tsx    → テーブルの1行（レコメンドボタン含む）
```
